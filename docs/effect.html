Developer Brief: Custom Game Hover Effects

THE CABINET launcher uses a decoupled UI architecture. Instead of hardcoding animations inside the core launcher (app.py), the launcher dynamically scales by auto-importing custom hover effects directly from each game's folder.

When a user hovers over your game card via touch, mouse, keyboard, or an arcade gamepad, the launcher looks for a file named effect.py inside your game's directory and executes it.

🏗️ Architecture Overview

The system isolates launcher drawing logic from game mechanics. Instead of keeping static checks inside the main menu loop, the launcher dynamically exposes the active card's rendering surface and bounding boxes directly to external scripts.

📁 root directory
│
├── 📄 app.py (Core Launcher)
│
└── 📁 games
    └── 📁 your_game_folder
        ├── 📄 config.json  (Metadata)
        ├── 📄 game.py     (Main Gameplay Payload)
        └── 📄 effect.py   (Menu Hover Animation)  <-- Your custom effect

🛠️ The effect.py Specification

To make a custom menu animation work with THE CABINET's auto-import layout, your effect.py file must contain a function named exactly draw_effect that accepts three specific arguments:

def draw_effect(screen, rect, ticks):
    """
    Renders a custom menu hover animation inside or around the game card.
    
    Arguments:
        screen (pygame.Surface): The master window display surface.
        rect (pygame.Rect):     The current bounding box of the selected menu card.
        ticks (int):            A continuously running frame counter for animations.
    """
    pass

Understanding the Arguments:

  • screen: The surface to draw your elements onto.
  • rect: The exact position and dimensions of the menu card. Because users can scroll kinetically or select via gamepads, rect.top, rect.bottom, rect.left, and rect.right will change dynamically. Always base your positioning coordinates on this rect object.
  • ticks: A standard integer that increments by 1 every frame (60 fps). Use this parameter instead of keeping track of separate timing variables to run time-based mathematics (like math.sin() or modulo operations %).

📋 Step-by-Step Implementation Guide

Follow this protocol to construct and bundle a new effect:

Step 1: Isolate Rendering Boundaries

To keep the main launcher text clean and layout margins consistent across games, constrain your custom drawings to the inside bounds of the card. Use Pygame's built-in clipping feature to guarantee drawings don't leak out during rapid menu scrolling:

import pygame
import math

def draw_effect(screen, rect, ticks):
    # Lock drawing strictly inside the game card's boundaries
    screen.set_clip(rect)
    
    # ... Your drawing logic goes here ...
    
    # CRITICAL: Always release the clip at the end so the rest of the launcher renders!
    screen.set_clip(None)

Step 2: Build Relative Animations

Do not use absolute coordinates like (500, 300). Use positions relative to the passed rect argument so that the effect naturally moves along with the launcher's kinetic scrolling:

    # Calculate a safe workspace area on the right side of the card
    workspace_w = 60
    render_x = rect.right - workspace_w - 20
    render_y = rect.centery

Step 3: Run Cycle Mathematics

Use the ticks parameter to execute loops or smooth transitions. For instance, to calculate a bouncing loop that repeats every 60 frames (1 second):

    loop_timeline = ticks % 60
    normalized_progress = loop_timeline / 60.0  # Moves from 0.0 to 1.0 smoothly

💡 Reference Boilerplate Template

Below is a production-ready template showing a complete effect.py workflow that draws a pulsing radar ring on the right edge of a game card:

import pygame
import math

def draw_effect(screen, rect, ticks):
    """Draws a flashing neon radar ring relative to the card's right boundary."""
    
    # 1. Protect layout canvas with clipping mask
    screen.set_clip(rect)
    
    # 2. Establish anchoring anchors
    center_x = rect.right - 50
    center_y = rect.centery
    
    # 3. Create timing logic using the master frame clock
    pulse_cycle = (ticks * 2) % 100
    radius = int((pulse_cycle / 100.0) * 35) + 5
    alpha = max(0, 255 - int((pulse_cycle / 100.0) * 255))
    
    # 4. Render alpha transparencies using a workspace canvas
    temp_surf = pygame.Surface((100, 100), pygame.SRCALPHA)
    pygame.draw.circle(temp_surf, (0, 240, 255, alpha), (50, 50), radius, 2)
    
    # 5. Composite back onto launcher display matrix
    screen.blit(temp_surf, (center_x - 50, center_y - 50))
    
    # 6. Release clip to preserve master launcher frame stack
    screen.set_clip(None)
